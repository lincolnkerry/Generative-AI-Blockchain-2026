## Submission workflow

1. Create a public GitHub repository for the team project.
2. Use the required repository structure from the main README.
3. Keep the 7-day usage log updated during the project period.
4. Submit the team repository link by pull request or by the course submission channel.
5. Final submission must include code, slides, paper/report, usage log, and demo video.

## Required final links

Each team should provide:

- Team repository URL
- Paper/report URL or path
- Slides URL or path
- Demo video URL
- 7-day usage log path
