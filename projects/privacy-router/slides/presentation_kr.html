02

Problem

AI 에이전트가 민감 데이터를 외부 LLM으로 전송합니다

문제

AI 에이전트는 민감 데이터를 외부 LLM으로 전송합니다

기존 솔루션(OpenAI Privacy Filter)은 영어/미국 PII(SSN, ZIP) 중심. 한국 RRN, +82 전화번호, 연구 맥락을 이해하지 못합니다. 모두 공개 또는 모두 차단만 가능합니다.

예시

주민등록번호 유출

주민등록번호(13자리) 같은 PII는 기존 LLM 가드레일이 놓칩니다. 정규표현식은 맥락을 이해하지 못합니다.

예시

미공개 연구 유출

연구 아이디어, 연구실 예산, 제출 계획 — 경쟁 정보는 정규식으로 탐지 불가능합니다.

결론

Privacy Router: 맥락 인식 프롬프트 방화벽

탐지, 마스킹, 라우팅 — 민감 데이터는 로컬에 안전하게, 안전한 쿼리만 클라우드로. 모든 사용자를 위한 데이터 주권.

Engineering · Policy Engine 6 / 15
07

Router

순수 실행 레이어 (LLM 없음) — 결정 테이블, UID 기반 마스킹, 하이드레이션

Route 1

External API

민감 스팬을 플레이스홀더로 대체 → 클라우드 LLM 전송 → 응답에서 원본 복원

mask_text() → Cloud LLM → hydrate_text()
Route 2

Local API

프롬프트 자체가 민감 정보에 관한 것일 때. 전체 처리가 로컬에서.

process_locally() → DGX Spark GB10
Route 3

사용자에게 질의

시스템이 결정을 내릴 수 없을 때. 사용자에게 확인 질의.

409 → needs_input → 사용자 선택
마스킹 컨트랙트

결정론적 마스킹: UID 기반 플레이스홀더 (RESIDENT_REGISTRATION_NUMBER#1) 사용. 원본은 SQLite에 암호화 저장. 마스킹 → LLM → 복원 과정에서 원본이 클라우드에 절대 노출되지 않습니다.

v2 — Contextual Reasoning

질문 기반 추론

Extractor가 모든 문장에 두 가지 맥락 추론 질문을 적용합니다:

"이 문장이 내일 신문에 실리면, 경쟁사가 이득을 볼까?"
"출판 전에 공개되면, 연구자가 피해를 볼까?"

이 질문들은 SLM이 키워드에 의존하지 않고 민감 정보를 탐지할 수 있게 합니다. 모델은 의미를 이해합니다 — "TSMC 3nm 공정 채택" 같은 문장에도 "비밀"이나 "기밀"이라는 명시적 키워드 없이 사업 기밀로 분류됩니다.

Privacy & Security

데이터 흐름과 위협 모델

데이터 흐름

민감 데이터는 로컬에 머무름

원본 텍스트 → 로컬 SLM에서만 처리
마스킹된 텍스트만 클라우드 LLM으로 전송
원본은 SQLite에 암호화 저장
복원은 로컬에서만 수행
위협 모델

보장된 보안 속성

✓ 민감 데이터 제로 유출 — 마스킹 없이는 외부 전송 불가
✓ 결정론적 마스킹 — 동일 입력 → 동일 마스킹 (재현 가능)
✓ 감사 가능 — 모든 결정이 로그에 기록
⚠ SLM 환각 위험 — 탐지 모델이 놓칠 수 있음 → Critic 패스로 완화
핵심 보안 원칙

"마스킹된 데이터만 클라우드로" — Privacy Router는 민감 데이터가 외부 LLM에 도달하는 것을 구조적으로 방지합니다. 사용자 동의 없이는 어떤 민감 정보도 외부로 전송되지 않습니다.
