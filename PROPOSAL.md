# Privacy Router: A Sovereignty-First Butler for Korean Researchers

*GenAI & Blockchain — Term Project Proposal · DH. Kim & M. Saadati · Type (iv) Privacy-Critical Agent · 2026-05-25*

---

## 1. Problem and thesis

Slide A.2 of L1: *"A Private AI is a routing layer that knows what must stay home."* OPF is US-centric (SSN, ZIP); it misses Korean RRN, `+82` phones, and institutional IDs. Worse, agent frameworks draft emails and autofill forms autonomously, packing invisible secrets into frontier LLM calls. A butler replying to *"send me your resident ID"* will silently ship your **RRN** to Claude. A document agent pulling lab notes will forward **unpublished data** without warning.

We build **Privacy Router**: an on-device **Extractor → Judge → Router** pipeline that intercepts every agent-generated prompt before egress. A Small Language Model (SLM; Qwen3.6 9B, Ministral 3 3B, or EXAONE 3.5 1.2B) surfaces secrets the user never realized were there — not just PII, but also **business decisions**, **research ideas**, and **pre-publication findings**. A policy-driven Judge decides whether the prompt is safe for external APIs or must stay local.

## 2. System architecture

Three layers, one custom MCP server in the middle.

### 2.1 Extractor (SLM-based detection)

Uses an SLM to detect sensitive information through **contextual analysis**, not pattern matching. The SLM generates **free-form category tags** in SCREAMING_CASE format based on its understanding of the content.

**Tag generation rules:**
- Tags are generated by the SLM, not from a fixed list
- Must be in SCREAMING_CASE (e.g., `MERGER_DECISION`, `NOVEL_ALGORITHM`)
- Must be descriptive enough for hydration (reversible placeholder substitution)

### 2.2 Judge (Two-axis policy engine)

The Judge evaluates extraction records using **two axes**:

**Axis 1: Does this need masking?**
- If the input contains sensitive information (PII, business secrets, research secrets) → Yes
- If no sensitive information detected → No → **safe for external API**

**Axis 2: Is this query meaningful after masking?**
- If the query is ABOUT the sensitive data (e.g., "내 주민등록번호가 뭐야?") → No → **must use local API**
- If the query just CONTAINS sensitive data as context (e.g., "이 아이디어를 바탕으로 실험 설계를 도와줘") → Yes → **mask and send to external API**

| Axis 1 (마스킹 필요) | Axis 2 (마스킹 후 유의미) | 결과 |
|---------------------|-------------------------|------|
| No | - | 외부 API 사용 가능 |
| Yes | Yes | 마스킹 후 외부 API 전송 |
| Yes | No | 로컬 API 사용 |

### 2.3 Router (Route selection + masking)

Routes requests based on the Judge's classification:
- **`external_api`**: Mask sensitive spans with placeholders, send to cloud LLM, hydrate response
- **`local_api`**: Process entirely on local LLM (DGX Spark GB10, qwen3:14b)

**Masking & Hydration**: For queries that remain meaningful after masking:
1. `mask_text()`: Replaces sensitive spans with stable placeholders (`[TAG_NAME#N]`)
2. Cloud LLM processes the masked text (never sees original values)
3. `hydrate_text()`: Restores original values in the response before returning to user

## 4. MCP plan

Three free servers + two custom servers.

* **filesystem** (sandboxed `~/workspace`) — read paper PDFs, draft notes locally; sensitive content never leaves the sandbox.
* **fetch** (`uvx mcp-server-fetch`) — pull arXiv abstracts and public regulation pages; only invoked on non-sensitive class.
* **memory** — persist routing decisions and curated notes across sessions; small, auditable local SQLite KV store.
* **privacy-router** (custom MCP, FastMCP, stdio) — three tools:
  * `classify(text: str)` → `{class, domain, rationale, tool_calls}` — runs Extractor → Judge; returns model-generated labels, reasoning, and extracted spans.
  * `redact(text: str)` → `str` — strips matched spans before any global LLM call (mixed class).
  * `route(text: str)` → `{model, endpoint, rationale}` — returns chosen worker/orchestrator + audit-log entry.

**Threat model.** `filesystem`: reads only `~/workspace` (no `/`, no `~`); injected file content treated as untrusted. `fetch`: confirm on first new domain; results never auto-acted upon. `memory`: local SQLite, no remote write. `privacy-router`: classification is model-driven, so a malicious description in another MCP server cannot force misclassification; the Judge retains final authority.

## 5. Three OpenClaw advantages addressed

**Privacy by default** — the router enforces the sensitive/public split before any egress; every decision is locally audited. **Cost transparency** — two-tier stack with token meter; target ≤ $10/month/user, with ~95% of calls on local or Haiku. **Customization** — the Korean-credential detection and personal/business taxonomy ship as a versioned skill; `SOUL.md` declares the router as the butler's identity.
